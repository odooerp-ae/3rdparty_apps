User Guide

1. Installation & Configuration:

  • Install the 'Maqsam Integration' module from Odoo Apps.
  • Navigate to `Settings > General Settings`.
  • Scroll down or look for the "Maqsam Integration" section.
  • Enter your `Maqsam Access Key` and `Maqsam Access Secret`.
  • Verify the `Maqsam API Base URL (v1)`, `Maqsam API Base URL (v2)`, and `Maqsam Portal Base URL`. Default values are usually sufficient.
  • Click "Save".

2. Agent Synchronization:

  • Go to `Maqsam API` menu in the Odoo sidebar.
  • Select `Maqsam Agents API`.
  • In the list view, click the "Fetch Agents" button to synchronize agent data from Maqsam to Odoo. This will create or update agent records with their latest status and details.

3. Initiating Calls (Click-to-Call):

  • To initiate a call from a pre-existing contact in Odoo:
    • Go to `Maqsam API > User`.
    • Create a new record or select an existing one. Ensure the `Contact` and `Phone Number` fields are populated.
    • Click on the "Call" button next to the contact in the list view (available in `Maqsam API > Call API` which shows "List of Contact").
  • To use the Maqsam Dialer for direct calls:
    • Go to `Maqsam API > Auto Dial API`.
    • Create a new record.
    • Enter the `Phone Number` you wish to dial. Optionally, enter a `Caller Name`.
    • Click "Auto Dial" in the list view to open the Maqsam web dialer in a new tab.

4. Maqsam Portal Auto-Login:

  • Go to `Maqsam API > Maqsam Auto Login`.
  • Create a new record.
  • Enter the `User Email` corresponding to a Maqsam agent.
  • Click the "Get Auth Token" button. This will retrieve a temporary authentication token from Maqsam. The `Auth Token` field will be populated.
  • Click the "Auto Login" button to be securely redirected to the Maqsam portal without needing to enter credentials manually.

Frequently Asked Questions

How do I get my Maqsam API Access Key and Secret?
You can obtain your API Access Key and Secret from your Maqsam account settings, usually under an "Integrations," "API Credentials," or "Developer" section. If you cannot find them, please contact Maqsam support for assistance.
Why are my calls not being initiated from Odoo?
Please ensure that:
  • The Maqsam API Access Key and Secret are correctly entered and saved in Odoo `General Settings > Maqsam Integration`.
  • The contact record in Odoo has a valid and correctly formatted phone number.
  • The Odoo user attempting to initiate the call is properly mapped or has permissions if relevant to your Maqsam setup.
  • Your Maqsam account is active and has calling capabilities.
  • There are no network connectivity issues preventing Odoo from reaching the Maqsam API. Check Odoo server logs for specific error messages related to Maqsam API calls.
  • Ensure the Maqsam API Base URLs in settings are correct. Default values are usually fine, but verify if you have a custom Maqsam setup.
Does this module log detailed call records (duration, recordings) in Odoo?
This module primarily logs the *initiation* status of the call within Odoo for tracking purposes. For comprehensive call details such as duration, call recordings, full call history, and analytics, you would typically refer to your Maqsam platform's dashboard and reporting features.
Is this module compatible with Odoo Enterprise and Community versions?
Yes, this module is developed to be compatible with both Odoo Community and Enterprise editions for the specified Odoo versions (e.g., Odoo 16, 17, and 18 as mentioned in the description).
How frequently is the Maqsam agent status (online, offline, busy) updated in Odoo?
The agent status synchronization is triggered by clicking the "Fetch Maqsam Agents" button in `Maqsam > Maqsam Agents`. It is not a continuous real-time push from Maqsam unless specified otherwise. For an up-to-the-minute status, you may need to re-fetch or rely on the Maqsam portal itself. Future versions might include automated periodic syncing.
What happens if an agent exists in Maqsam but not in Odoo, or vice-versa, during synchronization?
When you "Fetch Maqsam Agents":
  • If an agent exists in Maqsam but not in Odoo, a new agent record will typically be created in Odoo under `Maqsam Agents`.
  • If an agent exists in Odoo and Maqsam, their details in Odoo (like status, communication capabilities) will be updated based on the information from Maqsam.
  • Agents existing only in Odoo (e.g., manually created and not linked to a Maqsam agent) will likely remain untouched by the fetch process unless the module has a specific cleanup or mapping feature for such cases.
Can I create or delete Maqsam agents directly from Odoo?
The module focuses on synchronizing existing Maqsam agents into Odoo for visibility and enabling communication features. Agent creation and deletion are typically managed directly within the Maqsam platform to ensure all necessary Maqsam-specific configurations are handled correctly. Odoo provides a view and management layer for these synced agents.
How does the "Maqsam Dialer Auto-Launch" feature work?
When you use this feature (e.g., from `Maqsam > Dialer Call` by entering a number and clicking "Create Dialer Call"), the module constructs a specific URL that, when opened, launches the Maqsam web dialer interface in a new browser tab with the target phone number pre-filled, ready for you to initiate the call via Maqsam.
Is the "Seamless Maqsam Portal Auto-Login" secure? How long is the token valid?
Yes, the auto-login feature is designed to be secure. It generates a temporary authentication token specifically for the Maqsam portal. These tokens are typically short-lived to minimize risk. The exact validity period is determined by Maqsam's API and security policies. You should not share these tokens manually. The process involves Odoo securely requesting a token from Maqsam and then using it to redirect your browser.
Do I need an active Maqsam subscription to use this module?
Yes, this Odoo module is an integration tool. You need an active Maqsam account with the necessary calling features and API access enabled to use the functionalities provided by this module. The module connects Odoo to your existing Maqsam service.
What are the `Maqsam API Base URL (v1)`, `Maqsam API Base URL (v2)`, and `Maqsam Portal Base URL` settings for?
These URLs are the specific web addresses that Odoo uses to communicate with the Maqsam platform:
  • Maqsam API Base URL (v1 & v2): These are the endpoints for the Maqsam Application Programming Interface. Odoo sends requests to these URLs to fetch agent data, initiate calls, etc. Different versions (v1, v2) might be used for different functionalities.
  • Maqsam Portal Base URL: This is the main web address for accessing the Maqsam user portal. It's used for features like auto-launching the dialer or auto-logging into the portal.
The default values provided in the module are generally correct for standard Maqsam cloud deployments. You would only need to change these if you have a dedicated or private Maqsam instance with different URLs, or if Maqsam updates these endpoints in the future.
Where can I find support if I encounter issues with this module?
For issues related to the module's functionality within Odoo, its configuration, or errors specific to the integration, you should typically contact the developer or vendor from whom you acquired the module. For issues related to your Maqsam account, call quality, Maqsam portal access, or Maqsam API availability, you should contact Maqsam support.
